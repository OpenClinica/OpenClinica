3.3 Rules Help

This section allows you Create Rules for Cross Field and Cross form edit checks and can do advanced validations. Rules are run on CRF variables and are created in an XML file. You can create rules, import them and run them in batch.

All Rules are ordered through their Study Event Definition. You can choose to reorder the list of Rules by toggling the ascending or descending arrow located next to the Study Event Definition header. Similarly, you can also reorder the Rule list using the CRF Name, Group Label, or Item Name, and toggle their respective ascending or descending arrows.

With the Find function, located at the top of the Rule list, you can search for a set of Rule based on the Study Event Definition, CRF Name, Group Label, or Item Name.

In addition to viewing all the Rules, you are also presented with a link to Import Rules Assignments as well as a link to Test Rules.

The Manage Rule Assignments matrix displays the following columns:

1. Study Event Definition This cell indicates the Study Event Definition which is affected by the rule.
2. CRF Name This cell indicates the CRF Name which is affected by the rule.
3. Group Label This cell indicates the Group Label which is affected by the rule.
4. Item Name This cell indicates the Item Name which is affected by the rule.
5. Rules This cell indicates the name of the rule(s).
6. Action Type This cell indicates the Action Type of the rule. It could be FILE_DISCREPANCY_NOTE or EMAIL.
7. Actions (depending on role)

View

Clicking on the View icon will bring you to the details of this rule.

Run

Clicking on the Run icon will run the rule as specified.

Remove

To remove a rule assignment click the Remove icon.

Restore

To restore a rule assignment click the Restore icon.

Audit Logs

Clicking on the Audit Logs icon will open the Audit Log for Rule Assignment page with more details.
